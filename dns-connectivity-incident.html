DNS Connectivity Incident Resolution

Root Cause Analysis • Authentication & Name Resolution Troubleshooting

Incident Overview

Investigated and resolved DNS-related connectivity issues affecting two domain-joined workstations. Users were able to ping external IP addresses (e.g., 8.8.8.8) but were unable to resolve domain names such as google.com, indicating a name resolution failure.

Initial Symptoms

  • Successful ping to external IP addresses
  • Failure when pinging domain names
  • Intermittent domain connectivity issues

Root Cause Analysis

  • Identified incorrect DNS server configuration on affected machines
  • Confirmed devices were not properly pointing to the Domain Controller for DNS
  • Verified issue through network adapter inspection and resolution testing

Resolution

  • Obtained local administrator privileges
  • Manually configured network adapter to use the correct internal DNS server
  • Validated resolution using domain name testing and connectivity checks
  • Restored full network and authentication functionality

Skills Demonstrated

  • DNS troubleshooting and name resolution diagnostics
  • Root cause analysis
  • Windows network configuration
  • Authentication workflow understanding
  • Minimized user downtime through targeted remediation
