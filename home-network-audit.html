Kali Linux • Nmap • Internal Enumeration

Project Overview

Conducted an authorized internal security assessment of a private Class C network (192.168.X.0/24) using Kali Linux in a bridged VMware environment. The objective was to identify active hosts, enumerate exposed services on the gateway, and assess potential hardening opportunities.

Tools Used

  • Kali Linux (VMware - Bridged Mode)
  • Nmap
  • Linux networking commands (ip a, ip route)

Methodology

  • Identified network range using ip a and ip route
  • Performed host discovery: nmap -sn 192.168.X.0/24
  • Enumerated gateway services: nmap -sS -sV -O 192.168.X.1
  • Analyzed exposed services and classified risk levels

Findings

Port Service Description Risk Level
53 DNS (dnsmasq) Internal name resolution service Low
80 HTTP Router admin interface (unencrypted) Medium
443 HTTPS Secure router admin interface Low
49152 UPnP Automatic port mapping service Medium–High

Security Analysis

No Telnet (23) or SSH (22) services were exposed. The primary hardening opportunity identified was the presence of an active UPnP service and an HTTP management interface alongside HTTPS. Overall exposure was limited to internal services within the subnet.

Recommendations

  • Disable UPnP if not required
  • Restrict router management to HTTPS only
  • Confirm remote administration is disabled
  • Maintain firmware updates
